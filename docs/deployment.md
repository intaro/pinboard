# Запуск и эксплуатация

## Локальный запуск

Типовой порядок:

1. установить зависимости;
2. настроить `.env.local`;
3. собрать фронтенд;
4. подготовить базу данных;
5. запустить приложение через веб-сервер или Symfony CLI.

### Установка зависимостей фронта

```bash
pnpm install
```

### Сборка фронта

```bash
pnpm build
```

## Служебные задачи

- агрегация данных выполняется командой `aggregate`;
- регистрацию cron можно получить через `register-crontab`;
- для работы с пользователями есть команды миграции между файловым и БД-хранилищем.

## Пример crontab для агрегации

```cron
*/15 * * * * cd /var/www/pinboard/current && php bin/console aggregate --no-interaction >> /var/log/pinboard/aggregate.log 2>&1
```

Этот пример запускает агрегацию каждые 15 минут и пишет вывод в лог. Пути в строке можно адаптировать под конкретное окружение.

## Фронтенд

После изменений в `assets/` пересобирай фронтенд, чтобы синхронизировать исходники и `public/build`.

## Docker

Если используется Docker-окружение, ориентируйся на `Makefile` и локальные команды проекта.
